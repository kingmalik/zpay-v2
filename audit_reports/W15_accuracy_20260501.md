## HIGH Violations — ED Accounting Invariant (All Same Root Cause)

All 36 HIGH violations are `ED_GROSS_NET_DEDUCTION_MISMATCH`. The ED PDF ingest stores WUD/RAD deductions inside `gross_pay` (as `gross_pay - net_pay` gap) but does NOT write the corresponding amount into the `deduction` column. The audit rule expects `gross_pay == net_pay + deduction`; instead it finds `deduction=0` while `gross_pay > net_pay`.

**This does NOT affect driver pay.** `z_rate` (driver pay) is set from the service rate table, not from `gross_pay` or `deduction`. Maz's margin (WUD+RAD) is correctly implicit in `gross_pay - net_pay`, just not surfaced in the `deduction` column.

**Root cause:** `pdf_reader.py` needs to write `deduction = gross_pay - net_pay` (i.e. WUD+RAD total) when parsing ED PDFs, per the `business_ed_wud_rad.md` memory note.
