## Methodology

- **Corpus:** 1,100 FirstStudent/FirstAlt emails (JSON, `{id, date, from, subject, body}`), spanning **2024-10-15 to 2026-07-22**. Parsed locally with Python — no email was sent, no thread was replied to.
- **Primary search:** subject line matching `complaint` (case-insensitive) → **45 hits**, spanning 2025-04-01 to 2026-06-17. This is FirstAlt's own formal complaint-report template in the majority of cases, so it's a clean signal.
- **Secondary search:** body text matching `complaint` where the subject didn't → **33 hits**, manually scanned for anything the subject-only search would have missed. Most were boilerplate ("Report / Complaint Completed By:" forwarded under a non-"complaint" subject, already counted via a companion thread) or unrelated (a driver's own comment that a parent "wasn't happy," a mention of a *prior* complaint). **3 were material and not otherwise captured** — Ashraf Mohamed's 3rd-complaint escalation email, Safiullah Faqiri's parent safety letter, and the Seude Mohammed Adem drunk-at-pickup allegation — all folded into the ledger above because their severity outweighs the fact that "complaint" wasn't in the subject line.
- **Threading:** FirstAlt/Acumen email chains quote the entire prior thread on every reply, so a single incident often produces 3–10 separate messages. Each *incident* is one row above; the reply chatter (confirmations, thank-yous, tangents like a lost ID or a vehicle repair invoice) was read but not double-counted as separate complaints.
- **Cross-ref — reliability tiers:** pulled live from prod (`GET /api/data/reliability/tiers` via the same service-key pattern as `zpay-kpis.sh`) — this worked, and every shortlisted driver above is annotated with their live tier. All five came back "trusted." That gap — zero dispatch friction, real complaint history — is itself the finding worth sitting with.
